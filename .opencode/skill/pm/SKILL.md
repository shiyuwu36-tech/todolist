# PM Skill（PRD 与验收标准）

## 交付物
- PRD（1-2页）：目标用户、范围（MVP/可选）、用户故事、数据字段、非功能需求、测试计划
- 验收标准：每条必须可测试（手动步骤或自动化测试能验证）
- PRD 自检清单：逐条检查“可测/无歧义/不超范围”

## 规则
- PRD 未完成并自检通过前：禁止写任何代码
- 每个功能点必须对应验收标准编号（A1/A2…）
- 不清晰之处要写“假设 Assumptions”，并纳入 PRD
- 必须覆盖：前端页面、后端接口、持久化存储（服务重启后数据保留）、附件上传+下载（重启后仍可下载）、可在公司办公本运行（启动简单、依赖少、README清晰）

## 输出结构（固定）
1) 背景与目标
2) MVP 范围（必须做）/ 可选项（可不做）
3) 用户故事（US1…）
4) 数据模型字段（Task/Attachment）
5) API 列表（端点/请求/响应/错误码）（概要即可，细节交给技术方案）
6) 验收标准（A1…）
7) 非功能需求（性能/可访问性/兼容性/安全与隐私/部署可运行性）
8) 测试计划（与 QA 对齐）
9) PRD 自检清单（勾选项）

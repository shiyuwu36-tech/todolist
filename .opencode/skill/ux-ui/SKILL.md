# UX/UI Skill（文字版 UI Spec）

## 交付物
- UI Spec（文字版）：页面结构、组件清单、交互流程、状态设计
- 设计 tokens：间距（8px体系）、圆角、字体层级、阴影、暗色模式规则
- 可访问性要点：键盘操作、focus ring、aria 文案

## UI Spec 必含模块
- 顶部工具栏：搜索、过滤、排序、新建按钮、主题切换
- 任务列表：任务卡片（标题/优先级/负责人/dueDate/时效标签/完成状态）
- 任务编辑：Modal 或 Drawer（表单校验、Enter 提交、Esc 关闭）
- 删除确认：Modal（危险操作）
- 附件：任务详情区/编辑区的“附件列表 + 上传按钮”
  - 上传：选择文件→显示进度/loading→成功 toast→列表出现新附件
  - 失败：错误提示 + 重试按钮
  - 下载/预览（若做可选）：点击附件打开/下载
- 反馈：toast（新增/保存/删除/上传成功/上传失败/错误）

## 状态设计（必须）
- empty：无任务时的引导文案 + CTA
- loading：初始化加载（前端取数/后端请求中）
- error：API 请求失败/上传失败/数据异常时的提示与重试

## 输出格式
- 页面草图（文字）
- 组件清单（组件名→职责→props）
- 交互规则（点击/键盘/校验/反馈/上传流程）
